Q3 · 가장 인상 깊었던 순간 (선택 입력 · 6명 응답)
클래스에서 가장 인상 깊었던 순간이나 내용은 무엇인가요?
응답 01
비개발자, 초보자 입장에 대한 공감을 많이 해주시면서 설명해주시던 모습이 너무 좋았다.
응답 02
자연어로 서비스를 플래닝하는 과정이 인상적이었습니다.
응답 03
mcp, extension, skill 등 클로드와 VS Code를 더 잘 쓸 수 있는 방법을 알게 되었습니다.
응답 04
반복되는 업무에 대해 프로그램을 만들어 자동화할 수 있다는 것에 아주 인상 깊었습니다.
응답 05
직접 API를 연결해보고, 데이터를 추출해보는 과정이 전반적인 개발 플로우를 함께 알 수 있어서 좋았습니다.
응답 06
구현할 때 필요한 내용을 모두 알지 못해도 인터뷰 스킬을 통해 구체적인 기획안을 작성할 수 있다는 점이 인상 깊었습니다.
Q5-1 · 업무 활용 아이디어 (선택 입력 · 7명 응답)
어떤 것을 업무에 활용해볼 수 있을 것 같나요?
응답 01
업무용 사이트 생성
응답 02
필요한 업무의 빠른 자동화, 정리, 시각화
응답 03
활용해볼 수는 있지만 API 활용이 되지 않아 사실상 만들 수 있는 것이 거의 없을 것 같다..
응답 04
Confluence 문서로 활용
응답 05
개인정보 문제가 없다면 시도해보고 싶은 것들이 있는데 아직은 장벽이 있다고 봅니다.
응답 06
디자인 시스템을 자동으로 읽어서 문서를 정리하고, 어떤 방식으로 변경하면 좋을지 설계하고 Claude Code를 이용해 자동으로 디자인 토큰을 등록하는 등 다양한 업무에 사용이 가능합니다.
응답 07
대시보드 업데이트 자동화, 이메일 분류 자동화
Q7-1 · 추천/비추천 이유 (선택 입력 · 8명 응답)
추천하거나 추천하지 않는 이유를 한 줄로 적어주세요.
응답 01
내가 필요한 업무 비서를 만들 수 있을 것 같다.
응답 02
아예 모르는 분들한테는 아주 유용할 것 같습니다.
응답 03
쉽게 설명해주시고 개별 속도에 맞춰 진행해주셔서 좋았다.
응답 04
클로드로 꼭 업무와 직접적인 부분이 아니어도 응용할 수 있는 부분이 많다고 생각되기 때문에 기회는 무조건 있어야 한다고 봅니다.
응답 05
전사적으로 AI에 대한 이해도를 높이는 데 크게 도움이 되었던 것 같습니다.
응답 06
동일한 서비스를 개발하는 것이 아니라 각자 원하는 방향으로 직접 개발해볼 수 있었으면 더 좋았을 것 같습니다. 서비스 기획 단계에서 무엇을 고려해야 하는지 스스로 생각해야 업무 효율을 높이는 툴로 사용할 수 있다고 판단해서 아쉬움이 있었습니다.
응답 07
좀 더 공부해서 활용해보겠습니다. 좋은 기회 주셔서 감사합니다.
응답 08
아직 깃허브 등 비개발자에게 도전하기엔 어렵다. 그때 당시에도 수업에 따라가려고 노력했으나 어렵긴 하다.
